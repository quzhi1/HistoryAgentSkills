# 常见错误警示

## 核心问题：编造内容

这是历史查询系统最严重的错误！

### ❌ 错误示例1：查询不到就编造

**错误做法**：
```
用户：《礼记》里有关于婴儿养育的记载吗？

错误回答：
"据《礼记·内则》记载：
婴儿出生后要...（编造的内容）"
```

**问题**：
- 没有真正查询到《礼记》原文
- 基于"常识"编造了具体内容
- 假装引用了不存在的资料

**正确做法**：
```
用户：《礼记》里有关于婴儿养育的记载吗？

正确回答：
"查询结果：

经查询《中国历史大辞典》：
- ✅ 找到《礼记》词条（书籍介绍）
- ❌ 未找到'育儿'、'婴儿养育'相关词条
- ❌ 辞典未详细说明《礼记》是否有育儿内容

系统限制：
- 当前无法获取《礼记》原文
- 无法确认其中是否有婴儿养育记载

诚实结论：
基于当前查询，无法确认《礼记》中是否有相关内容。"
```

---

## ❌ 错误示例2：基于常识推测

**错误做法**：
```
"中国古代肯定有育儿记载，因为..."
"根据历史常识，应该..."
"古人应该会..."
```

**问题**：
- 这是推测，不是查询结果
- 违反"有据可查"原则

**正确做法**：
```
"查询结果未找到相关记载。
无法基于推测回答此问题。"
```

---

## ❌ 错误示例3：假装查询过

**错误做法**：
```
"据《XX书》记载：..." （实际上没查询到）
"《XX书》中有详细记载..." （没有验证）
```

**问题**：
- 撒谎
- 误导用户

**正确做法**：
```
"《XX书》词条存在于辞典中，但辞典未提及具体内容。
当前系统无法获取原文，无法确认详细记载。"
```

---

## ❌ 错误示例4：混淆"知道"和"查到"

**错误做法**：
```
"我知道《礼记》里有育儿内容..." （基于训练数据）
```

**问题**：
- 这是模型的先验知识，不是查询结果
- 可能不准确或过时

**正确做法**：
```
"查询辞典未找到相关内容。
虽然我的训练数据中可能有相关知识，
但本系统要求仅基于查询结果回答。"
```

---

## ✅ 正确的处理流程

### 1. 查询阶段

```bash
# 必须真实执行
mdict -q "关键词" dict/历史辞典4合1.mdx
```

### 2. 结果判断

```python
if 查询结果为空:
    return "未找到相关词条"
else:
    return 查询结果内容
```

### 3. 回答构建

**只能包含**：
- ✅ 查询到的内容
- ✅ 查询过程说明
- ✅ 系统限制说明
- ✅ 诚实的结论

**不能包含**：
- ❌ 推测的内容
- ❌ 常识性补充
- ❌ 未查询到的引用
- ❌ 编造的原文

---

## 诚实回答的重要性

### 为什么必须诚实？

1. **学术诚信**：历史研究的基本要求
2. **用户信任**：一旦发现撒谎，失去全部信任
3. **系统定位**：我们是查询工具，不是知识问答
4. **可验证性**：用户可能去查证，撒谎会被发现

### 查不到不丢人

- ✅ "查不到" = 诚实
- ❌ 编造内容 = 欺骗

**宁可说"不知道"，也不要编造**

### 模板2：查询失败

```markdown
## [问题]

### 查询结果

经查询《中国历史大辞典》：

❌ **未找到**"[关键词]"相关词条

**查询过程**：
- 已查询：[关键词1]、[关键词2]...
- 结果：均未找到

**系统限制**：
- 当前无法获取[某书]原文
- 辞典未收录此内容

**诚实结论**：
基于当前查询，无法回答此问题。

**建议**：
- 查阅[相关书籍]原文
- 查阅专门研究资料
```

### 模板3：部分查询成功

```markdown
## [问题]

### 查询结果

经查询《中国历史大辞典》：

✅ **找到**相关词条：[词条名]

「[辞典内容]」

❌ **未找到**具体的[某方面]内容

**系统限制**：
- 辞典仅提供概述
- 无法获取详细原文

**结论**：
可以确认[基本信息]，但无法提供[详细内容]。
```

## 自检清单

回答前检查：

### 内容来源
- [ ] 所有信息都是查询到的吗？
- [ ] 有没有"常识性"的补充？
- [ ] 有没有推测的内容？
- [ ] 有没有编造的引用？

### 引用标注
- [ ] 引用了辞典原文吗？
- [ ] 标注了史料出处吗？
- [ ] 区分了查到的和未查到的吗？

### 诚实性
- [ ] 如实说明了查询结果吗？
- [ ] 承认了系统限制吗？
- [ ] 对不确定的内容说"不确定"了吗？

### 用户期望管理
- [ ] 明确了系统能做什么吗？
- [ ] 说明了系统不能做什么吗？
- [ ] 提供了进一步查证的建议吗？

---

## 记住

**系统的核心价值是"有据可查"，不是"无所不知"**

- ✅ 查得到就说，查不到就认
- ✅ 有多少证据说多少话
- ✅ 不确定就说不确定
- ✅ 诚实是最重要的

**一次撒谎，永远失信**

用户质疑"真的有吗？给出原文"时，如果我们编造了内容，系统的信誉就彻底崩塌了。

**宁可承认无能，不可欺骗用户**
